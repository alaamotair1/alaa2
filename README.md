# مشروع بورتفوليو شخصي

هذا المشروع عبارة عن موقع بورتفوليو شخصي (Portfolio Website) مصمم لعرض الأعمال والمهارات، مع دعم كامل للغة العربية (RTL) وتصميم متجاوب.

## تركيب المشروع محليًا

لتركيب وتشغيل المشروع على جهازك المحلي، اتبع الخطوات التالية:

1.  **استنساخ المستودع (Clone the repository):**
    إذا كان المشروع على GitHub، يمكنك استنساخه باستخدام الأمر التالي في الطرفية:
    ```bash
    git clone [رابط المستودع هنا]
    cd PortfolioProject
    ```
    إذا كان لديك ملف `zip`، فما عليك سوى فك ضغط الملف في مجلد من اختيارك.

2.  **فتح المشروع:**
    بعد فك ضغط المشروع أو استنساخه، انتقل إلى المجلد `html`.
    ```bash
    cd html
    ```

3.  **عرض الموقع:**
    افتح ملف `index.html` في متصفح الويب المفضل لديك. يمكنك ببساطة النقر المزدوج على الملف، أو سحبه وإفلاته في نافذة المتصفح.

    يجب أن يعمل الموقع بشكل كامل محليًا، مع جميع الروابط والموارد (CSS, JavaScript, صور الـ placeholder) بشكل صحيح.

## كيفية نشر الموقع على GitHub Pages

GitHub Pages هي خدمة استضافة مجانية تمكنك من نشر مواقع الويب الثابتة مباشرة من مستودع GitHub الخاص بك. اتبع هذه الخطوات لنشر موقع البورتفوليو الخاص بك:

1.  **إنشاء مستودع جديد على GitHub:**
    -   اذهب إلى [GitHub](https://github.com/) وسجل الدخول إلى حسابك.
    -   انقر على زر `+` في الزاوية العلوية اليمنى، ثم اختر `New repository`.
    -   أعطِ المستودع اسمًا (مثلاً `my-portfolio` أو `[اسم المستخدم الخاص بك].github.io` إذا كنت تريد أن يكون الموقع متاحًا على نطاق فرعي خاص بك). تأكد من أنه `Public`.
    -   لا تقم بتهيئة المستودع بملف `README` أو `gitignore` أو ترخيص في هذه المرحلة.
    -   انقر على `Create repository`.

2.  **رفع ملفات المشروع إلى المستودع:**
    -   افتح الطرفية (Terminal أو Git Bash) على جهازك.
    -   انتقل إلى المجلد الرئيسي لمشروعك (PortfolioProject).
    -   قم بتهيئة Git ورفع الملفات:
        ```bash
        git init
        git add .
        git commit -m "Initial portfolio website setup"
        git branch -M main
        git remote add origin [رابط المستودع الذي أنشأته للتو، يبدأ بـ https://github.com/...]
        git push -u origin main
        ```

3.  **تفعيل GitHub Pages:**
    -   بعد رفع الملفات، ارجع إلى مستودعك على GitHub في المتصفح.
    -   انقر على تبويب `Settings`.
    -   في الشريط الجانبي الأيسر، انقر على `Pages`.
    -   تحت قسم `Branch`، اختر `main` (أو الفرع الذي رفعت عليه ملفاتك) من القائمة المنسدلة.
    -   اختر مجلد `html` كـ `/(root)` أو `docs/` إذا كنت قد وضعت ملفات HTML في مجلد `docs` (في هذا المشروع، ملفات HTML موجودة في `html/`).
    -   انقر على `Save`.

4.  **زيارة موقعك:**
    بعد بضع دقائق (قد يستغرق الأمر من دقيقة إلى 10 دقائق)، ستظهر رسالة في صفحة GitHub Pages تشير إلى أن موقعك منشور. ستجد الرابط المباشر لموقعك هناك (عادةً ما يكون `https://[اسم المستخدم الخاص بك].github.io/[اسم المستودع]/html/index.html` إذا كان المستودع ليس `[اسم المستخدم].github.io`).

    **ملاحظة هامة:** نظرًا لأن ملفات HTML الخاصة بك موجودة داخل مجلد `html/`، فإن الرابط المباشر لموقعك على GitHub Pages سيتضمن `html/` في المسار. على سبيل المثال: `https://yourusername.github.io/your-repo-name/html/index.html`.

## ملاحظات حول استخدام WowSlider (إذا تم استخدامه)

(هذا القسم سيتم تحديثه إذا قررت استخدام WowSlider بدلاً من Bootstrap Carousel.)

في حال تم استخدام WowSlider، سيتم تضمين ملفاته (CSS, JS, images) داخل مجلد المشروع. لضمان عمله بشكل صحيح، تأكد من أن المسارات في ملفات HTML تشير بشكل صحيح إلى هذه الموارد. عادةً ما يتطلب WowSlider بعض التكوين الأولي في ملف HTML حيث يتم استخدامه، والذي سيتم توفيره ضمن الكود.

**ملاحظة:** حاليًا، تم استخدام Bootstrap Carousel كما هو مطلوب، لذا هذا القسم غير مطبق.

## ملاحظات إضافية

-   **الصور والشعار:** تم تحديث المشروع بالصور التي تم توفيرها. تم استخدام الصورة الأولى لاستخراج لون الخلفية، والصورة الثانية كصورة جمالية في الصفحة الرئيسية، والصورة الثالثة كشعار للموقع. تم استبدال صور الـ placeholder في صفحة الأعمال (Portfolio) بصور ذات صلة من نتائج البحث.

-   **التحقق من النماذج (Form Validation):** يتم التحقق من صحة النماذج (Contact, Login, Signup) باستخدام Vanilla JavaScript على جانب العميل فقط. لا يتم إرسال البيانات إلى أي خادم. لربط هذه النماذج بـ backend، ستحتاج إلى إضافة كود JavaScript لإرسال طلبات `fetch` أو `XMLHttpRequest` إلى نقطة نهاية API مناسبة على الخادم الخاص بك.
    مثال لربط نموذج تسجيل الدخول بـ backend (في `js/validation.js`):
    ```javascript
    // داخل دالة submit الخاصة بنموذج تسجيل الدخول
    // ...
    if (isValid) {
        event.preventDefault(); // منع الإرسال الافتراضي
        const email = emailInput.value;
        const password = passwordInput.value;

        fetch('/api/login', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify({ email, password }),
        })
        .then(response => response.json())
        .then(data => {
            console.log('Success:', data);
            // هنا يمكنك التعامل مع استجابة الـ backend، مثل إعادة التوجيه أو عرض رسالة نجاح
            alert('تم تسجيل الدخول بنجاح!');
        })
        .catch((error) => {
            console.error('Error:', error);
            // هنا يمكنك التعامل مع الأخطاء، مثل عرض رسالة خطأ للمستخدم
            alert('حدث خطأ أثناء تسجيل الدخول.');
        });
    }
    // ...
    ```
